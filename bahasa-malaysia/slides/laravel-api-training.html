Hari 3

API Security

Peserta menambah authentication, token abilities, frontend token validation, dan rate limiting pada route group.

Halaman PDF 11-136 jam
Hari 3Frontend token + throttling + expiring Sanctum token + abilities
Pipeline security

Setiap request protected melalui beberapa lapisan semakan.

Client-> X-API-TOKEN?-> throttle:60,1-> auth:sanctum-> abilities-> Controller-> JSON

Frontend token tiada

Pulangkan 401 dengan error JSON yang jelas.

Terlalu banyak request

Laravel memulangkan 429 untuk melindungi API.

Token ditolak

Sanctum pulangkan 401 untuk token tiada atau expired, dan 403 untuk missing ability.

Peraturan securitySatu layer sahaja bukan keseluruhan sistem security
Route group security

Route grouping menjadikan rules security lebih jelas.

Public route

Route::post('/auth/login', [AuthController::class, 'login'])
    ->middleware(['frontend.token', 'throttle:login']);

Protected routes

Route::middleware([
    'frontend.token',
    'auth:sanctum',
    'throttle:api',
])->group(function () {
    Route::apiResource('users', UserProfileController::class)
        ->middlewareFor(['index', 'show'], 'abilities:profiles:read')
        ->middlewareFor('store', 'abilities:profiles:create')
        ->middlewareFor('update', 'abilities:profiles:update')
        ->middlewareFor('destroy', 'abilities:profiles:delete');
});
Detail Hari 3Login boleh public kepada user, tetapi bukan terbuka kepada frontend client yang tidak dikenali

Hari 3 - React auth

Protected browser CRUD calls perlukan dua API security headers.

Response login

{
  "message": "Login successful.",
  "data": {
    "token_type": "Bearer",
    "access_token": "1|plain-text-token",
    "expires_at": "2026-06-09T09:30:00Z",
    "abilities": [
      "profiles:read",
      "profiles:create",
      "profiles:update",
      "profiles:delete"
    ]
  }
}

Headers protected request

{
  'X-API-TOKEN': FRONTEND_API_TOKEN,
  Authorization: `Bearer ${token}`
}

Model performance

Optimize read, invalidate selepas write.

GET users->ability read->cache key->cache hit?
cache miss->UserProfile::with('projects')->simpan cache
POST/PATCH/DELETE->write ability->write model->clear cache

Amalan utama

  • paginate setiap list endpoint
  • guna eager loading untuk elak N+1 queries
  • masukkan page/search filters dalam cache keys
  • clear cache lama selepas write
